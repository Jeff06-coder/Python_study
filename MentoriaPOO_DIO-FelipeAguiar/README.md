# POO - Orientação a Objetos:
## Consiste em programar em uma forma mais organizada, fazendo códigos bem estruturados e legíveis para melhor manutenção.A Orientação a Objetos tem seus pilares funcionais, onde divide as funções do código para melhorar a leitura, etc...

## Herança:
### A herança consiste em fazer várias classes (dependendo), onde vai existir Class "mãe" e "filha", onde a 'mãe' da os parâmetros gerais (atributos e métodos) e as 'filhas' herdam esses parâmetros e funções dela, para ser mais fácil a manutenção e a reutilização dos códigos.Para repassar essas "leis" da Class mãe é necessário usar o "super().__init__(pMãe1, pMãe2)", existe uma maneira também de "privar" as Functions e Atributos da Class, utilizando "__" na frente deles.(É importante lembrar que se você privatizar algo dentro da Class, "isso" "só" poderá ser usado por essa Class)

## Encapsulamento:
### O encapsulamento trata-se de "separar" as "coisas" complicadas (partes lógicas do código ou até calculos mesmo) do código, onde você deseja fazer de uma forma para utilizar em um lugar especifico e depois que você for inclementar em outro lugar e for necessário modifica-la, você vai conseguir facilmente, pois voê já as tinha "dividido".

## Polimorfismo:
### É rescrever/reutilizar os métodos, é fazer um método ter mais de uma forma, onde em uma Class a function é definida de uma forma e em outra Class é definido de outra forma.

## Abstração:
### É tornar a Function subjetiva a como vai funcionar, faz as classes "filhas" obrigatoriamente inclementar os métodos dentro delas, escritos da mesma forma ("nome") da Class Super.
